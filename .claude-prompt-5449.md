## Objective

Implement the Muon optimizer for matrix-shaped parameters as an alternative to AdamW/SGD in `src/projectodyssey/training/optimizers/`.

## Motivation

Muon (Jordan et al. 2024, *"Muon: An optimizer for hidden layers in neural networks"*) has become a notable post-AdamW optimizer in 2024-2025. It applies Newton-Schulz iteration to orthogonalize the momentum buffer before each update, which on matrix-shaped parameters (linear/conv weights) consistently outperforms AdamW on language and vision tasks at comparable compute. Adopters include Kimi K2 and the NanoGPT speedrun line.

The grokking-experiment example landed under `examples/grok/lenet_emnist/` (see PR referencing this issue) currently uses the existing AdamW; adding Muon enables a "compare optimizers on the same grokking task" follow-up experiment, plus general-purpose use for paper reproductions where Muon now matters.

## Deliverables

- [ ] `src/projectodyssey/training/optimizers/muon.mojo` implementing `muon_step(params, gradients, momentum_buffer, lr, momentum, weight_decay, ns_steps=5)` with the same signature shape as `adamw_step`.
- [ ] Newton-Schulz orthogonalization helper (5-iteration variant from the paper) operating on a `Tensor[dtype]` matrix view.
- [ ] Hybrid-optimizer wiring guidance: Muon is documented to apply only to *matrix-shaped* params (conv kernels reshaped, linear weights). Embeddings, biases, scalar params should fall back to AdamW. Provide a `MuonParamFilter` helper or document the pattern.
- [ ] Unit tests under `tests/projectodyssey/training/optimizers/test_muon.mojo` covering: (1) shape correctness, (2) one-step descent on a quadratic, (3) Newton-Schulz convergence to an orthogonal matrix (singular values within 1e-5 of 1.0 after 5 steps).
- [ ] Smoke-train an existing example (LeNet/EMNIST is fine) to confirm convergence is at least as good as SGD baseline.

## Acceptance criteria

- Matches the `adamw_step` function/struct shape so it slots into existing training loops with minimal changes.
- Integrates with `CheckpointManager` (no special-case serialization needed).

## Out of scope

- Distributed multi-GPU Muon variants (Triton kernel, all-gather of orthogonalized momentum) — separate issue.
- ZeRO-style sharded state — separate issue.
- Embedding-specific Muon variants (the published recipe explicitly excludes embeddings from Muon).
